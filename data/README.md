# Data Directory

This directory holds all datasets used by DiagGen+.  
**None of these files are committed to Git** (see `.gitignore`).

---

## Folder structure

```
data/
├── raw/          ← Downloaded dataset files go here (unmodified)
├── processed/    ← Cleaned, tokenised, split datasets (generated by Phase 0 notebook)
└── synthetic/    ← VAE-generated augmentation samples (generated by Phase 2 notebook)
```

---

## How to download the dataset

### Option A — Kaggle (~25k records, recommended)

1. Install the Kaggle CLI if not already present:
   ```bash
   pip install kaggle
   ```
2. Place your `kaggle.json` API token in `~/.kaggle/kaggle.json`.
3. Run:
   ```bash
   kaggle datasets download -d kaushil268/disease-prediction-using-machine-learning -p data/raw --unzip
   ```
   > Expected file: `data/raw/dataset.csv` with columns `symptoms` and `prognosis`.
   > Rename `prognosis` → `disease` in the Phase 0 notebook.

### Option B — Hugging Face (1,065 records, faster to start)

```python
from datasets import load_dataset
ds = load_dataset("gretelai/symptom_to_diagnosis")
ds["train"].to_csv("data/raw/symptom_to_diagnosis.csv", index=False)
```
> Expected columns: `input_text` (symptoms) and `output_text` (disease).
> Rename in Phase 0 notebook: `input_text` → `symptoms`, `output_text` → `disease`.

### Option C — Symptom2Disease (Kaggle, 1,200 records)

```bash
kaggle datasets download -d niyarrbarman/symptom2disease -p data/raw --unzip
```
> Expected file: `data/raw/Symptom2Disease.csv` with columns `label` and `text`.
> Rename: `text` → `symptoms`, `label` → `disease`.

---

## Important notes

- All datasets are **publicly available** and contain **no real patient records (non-PHI)**.
- Synthetic records in `data/synthetic/` are generated by the Conditional VAE (Phase 2).  
  They are labelled `is_synthetic=True` and are **never presented as real data**.
- Do not commit any file in `data/raw/`, `data/processed/`, or `data/synthetic/` to Git.
